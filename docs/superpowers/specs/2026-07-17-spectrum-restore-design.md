# 光谱还原软件设计

## 目标

在 `huanyaunchazhi/restore` 中创建一个仅用于 Windows 的 PyQt6 光谱还原软件。

软件使用用户手动选择的 TXT 文件第一列作为完整横坐标，将 `compress_data` 生成的单列十六进制强度映射回真实采样位置，再使用 PCHIP 补齐其余强度。

实现不得修改 `compress_data` 下的任何文件。

## 范围

软件支持选择一个横坐标模板、批量选择一个或多个压缩文件、选择输出目录、执行还原并显示逐文件结果。

软件只支持 Windows，不提供 Linux 或 macOS 启动和打包脚本。

软件使用 Python 3.10+ 和 PyQt6，不依赖 NumPy 或 SciPy。

软件提供中文界面、Windows 启动脚本、PyInstaller 打包脚本、自动化测试和 Markdown 使用说明。

## 已选方案

项目采用独立实现方案。

项目在自身代码中复现压缩选点契约，并使用纯 Python 实现 PCHIP。

该方案不会引入对 `compress_data` 路径或安装状态的运行时依赖，也能控制 Windows 可执行文件体积。

测试使用固定样例验证选点结果与当前压缩规则一致。

## 项目结构

`compression_contract.py` 定义固定波数、邻域半径、降采样步长和索引选择函数。

`parsing.py` 解析模板横坐标和单列十六进制强度。

`interpolation.py` 实现 PCHIP、线性回退和数值校验。

`restoration.py` 负责索引映射、真实点覆盖、批量还原和原子写入。

`formatting.py` 负责强度的四位小数格式化。

`ui/main_window.py` 提供主窗口和用户操作。

`ui/worker.py` 在后台线程中执行批量还原并发送进度信号。

`main.py` 创建应用并启动主窗口。

核心模块不导入 PyQt6，以便独立执行单元测试。

## 模板输入

用户必须手动选择一个 TXT 文件。

程序逐行读取非空行，并只解析每行第一列。

分隔符支持英文逗号、中文逗号、空格和制表符。

每个非空行的第一列必须是有限数值，模板不得包含表头或非数值数据行。

横坐标不得重复，但不强制模板按升序排列。

程序同时保存第一列的原始文本和对应浮点数值。

输出第一列逐行使用保存的原始文本，因此不会重新格式化用户提供的横坐标。

模板第二列及后续列全部忽略。

## 压缩输入

用户可以一次选择一个或多个 TXT 文件。

每个非空行必须只包含一个 2、4、6 或 8 位十六进制整数。

十六进制解析不区分大小写，数值范围为 0 到 4294967295。

程序必须先将每行十六进制文本转换为十进制整数，例如 `012c` 转换为 `300`。

后续映射、插值、真实点覆盖和文件输出全部使用转换后的十进制强度，输出文件不得保留十六进制表示。

空文件、非法字符、非法长度或多列内容都会使当前文件失败。

单个文件失败不会中断其他已选文件。

## 压缩选点契约

独立实现必须复现 `compress_data/src/spectrum_compressor/compression.py` 当前的选点规则。

固定波数为 981、1386、2911、2578、2593 和 4134。

普通区域从索引 0 开始，每隔 35 个点保留一个点。

首点和末点始终保留。

对于位于模板波数范围内的每个固定波数，程序选择距离该波数最近的模板索引。

距离相同时选择较小索引。

程序保留中心索引及其左右各 9 个索引，并在边界处截断。

所有索引合并、去重并按原始索引升序排列。

压缩强度按文件行序依次映射到这些索引。

压缩强度数量必须与计算出的保留索引数量完全相等，否则当前文件失败。

## 插值算法

一个真实点时，程序直接输出该点，不执行插值。

两个真实点时，程序使用线性插值。

三个或更多真实点时，程序默认使用分段三次 Hermite 保形插值。

PCHIP 导数使用加权调和平均和单调性限制计算，避免在单调区间产生不必要的过冲。

插值前，程序按横坐标数值对真实点排序。

插值完成后，程序将结果恢复到模板原始行序。

如果 PCHIP 遇到非有限数值或数值计算失败，程序对当前文件自动改用线性插值并记录警告。

程序不执行范围外推。

在压缩契约匹配时，模板首尾索引都是保留点，因此全部模板横坐标都位于真实点覆盖范围内。

程序最后再次用解码后的压缩整数覆盖全部保留索引，确保插值无法修改真实点。

## 输出格式

默认输出目录为首个压缩文件所在目录下的 `Restored`。

用户可以在界面中选择其他输出目录。

每个成功文件输出为 `<压缩文件名主体>_restored.txt`。

输出使用 UTF-8 编码和 Windows 可正常读取的换行格式。

每行格式为 `横坐标原文,还原强度`。

真实保留点以解码后的整数原值输出。

插值点四舍五入到最多 4 位小数，并删除小数末尾的零和多余小数点。

程序先在目标目录写入临时文件，再原子替换正式输出，避免留下半成品。

## PyQt6 界面

主窗口提供模板路径、压缩文件列表和输出目录三个输入区域。

模板区域提供选择按钮和只读路径框。

压缩文件区域提供多选按钮、清空按钮和文件列表。

输出区域提供目录选择按钮和只读路径框。

窗口提供开始还原按钮、总体进度条、状态摘要和只读中文日志。

开始还原前，程序校验模板、压缩文件列表和输出目录是否齐全。

批量处理运行在后台工作对象中，避免阻塞窗口事件循环。

任务运行期间，程序禁用会改变输入的控件。

任务完成或失败后，程序恢复控件并显示成功数、失败数和每个失败原因。

## 错误处理

模板级错误会阻止整个批次开始。

模板级错误包括未选择文件、读取失败、空模板、非数值横坐标、非有限横坐标和重复横坐标。

文件级错误只影响当前压缩文件。

文件级错误包括读取失败、空文件、非法十六进制行、强度数量不匹配、插值失败和保存失败。

所有用户可见错误使用中文描述，并包含相关文件名和可执行的修正提示。

新项目的源代码、文档和配置统一使用 UTF-8，避免复制现有项目中的乱码字符串。

## 测试策略

开发采用测试驱动方式。

选点测试覆盖步长采样、首尾点、六个固定波数、左右 9 点、边界截断、重叠去重和距离相同时选择较小索引。

解析测试覆盖模板分隔符、原始横坐标文本、忽略后续列、非法横坐标、重复值和十六进制格式。

插值测试覆盖常量、线性、单调曲线、峰谷形状、两点回退、数值失败回退和真实点精确覆盖。

输出测试覆盖最多四位小数、末尾零删除、文件命名、UTF-8、模板原文保留和原子写入。

批量测试覆盖部分成功、部分失败和结果汇总。

PyQt6 冒烟测试在 offscreen 平台创建主窗口，验证控件初始状态、输入校验和任务完成后的状态恢复。

最终检查记录 `compress_data` 的 Git 差异，并确认该目录没有修改。

## 验收标准

应用可以在 Windows 上通过 PowerShell 脚本启动。

应用可以选择一个模板和多个压缩文件，并在后台完成批量还原。

每个输出文件的第一列与模板第一列原始文本逐行一致。

全部压缩强度都出现在重新计算出的真实索引上，且数值与十六进制解码结果完全一致。

输出强度全部使用十进制表示，不包含十六进制编码文本。

其他位置使用 PCHIP 或规定的线性回退生成，最多保留 4 位小数。

输入错误不会产生格式混乱或半写入的输出文件。

自动化测试和界面冒烟测试全部通过。

`compress_data` 下的任何文件都没有被修改。
